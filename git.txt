
1. git rm --cache fileName / git rm -r --cache . (删除所有)
        删除缓存区或分支中的对应文件，工作区保留但是不会被追踪（常见应用于配置.gitignore）

2. git rm fileName / git rm -r . (删除所有)
        同时删除缓存区、分支和工作区的对应文件

3. git reset --hard HEAD
    git reset HEAD e392d71(这个值是上一次add后的版本指针，由git返回)
        将工作区中所有文件回退到上一次commit的状态

4. git checkout -- git.txt
        将单个文件回退到上一次在工作区的状态(如果已经add或者commit，则需要配合reset)

5. git revert HEAD
